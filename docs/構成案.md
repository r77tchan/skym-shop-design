## 前提

- SPA は SEO的に弱い
  - Astro や Next などを使用したい

- モバイルアプリ版の作成などが必要ならフロントとバックを分離したい
  - 予定はないので Nextでもいい

## ホスティングサービス + DBサービス の問題点

- VPSより値段のコストは高い
  - 代わりに圧倒的楽
- 使用量の観点から、プリフェッチの無効化が検討される
- WebSocketのようなリアルタイム通信は利用できない
  - 予定はない

## VPSの問題点

- DBデータ、画像データの自前バックアップが必要
  - できれば外部にもバックアップしておきたい

- セキュリティアップデート、監視、ログ、復旧手順が必要
  - ディスク容量、バックアップを定期確認したい
  - サーバーが落ちた時に検知したい
  - 色々運用が大変

- 画像配信をVPSで行うと、帯域と表示速度が課題になる可能性もある
  - 結局、画像の保存と配信には R2 などを使用したい
  - ついでに画像のリサイズにも別サービス（Cloudflare Images / Image Transformations）を使用してもいいが、VPSなのでNextの画像最適化を無制限に使えそう
  - 結局、CDNキャッシュの為に Cloudflare などの CDN をVPSの前に置きたい

- メール配信、決済は外部サービスの利用が必須
  - 自前は大変（迷惑メールにならない為の工夫など）

## AWSやGCPの問題点

- 理解が浅い

- 従量課金が多い？
  - VPSより値段が安いという訳でもないかも

- 種類が多いので何とも言えないが、結局VPSと同じ問題を抱えるかも
  - 手間を軽くしたいなら結局、外部のDBサービスなどを使用することとなり、それなら初めからvercel + supabaseなどでいいのでは

## 使用検討サービス

| 目的               | サービス | 無料枠 | 費用             | 備考                                                                                                                   |
| ------------------ | -------- | ------ | ---------------- | ---------------------------------------------------------------------------------------------------------------------- |
| 画像保存先         | R2       |        | 無料枠でいけそう | リサイズの実施手段は別途必要。 S3、Cloudinaryなどが代替                                                                |
| メール送信         | Resend   |        | 無料枠でいけそう | SendGrid、AWS SESなどが代替                                                                                            |
| 決済               | Stripe   |        | 手数料3~4%程度   | 一応、日本の決済手段に多く対応したサービス（KOMOJU）や、手数料がもう少し安い（PAY.JP）、など他のサービスも存在している |
| 監視（エラー検知） | Sentry   |        |                  |
