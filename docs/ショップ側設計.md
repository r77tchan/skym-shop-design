# ショップ側設計

## URL

```txt
/
/items
/items?page=<page>
/items/<category_slug>
/items/<category_slug>?page=<page>
/item/<product_id>
/news
/news/<news_id>
/contact
/law
/privacy
```

未定義のURLはショップ側の404を表示する。

カテゴリのスラッグは固定値ではなく、管理画面の設定から追加、編集できる。

## ページング

- 商品一覧とカテゴリ一覧は、SEOのためページごとに一意のURLを用意する。
- 1ページ目は `page=1` を付けず、常に `/items` または `/items/<category_slug>` にする。
- 2ページ目以降は `?page=2` の形式にする。
- `page=1` にアクセスされた場合は、クエリなしのURLへcanonicalを向ける。
- 商品リスト下部には、次ページへの実リンクを用意する。

```html
<a href="/items/spoons?page=2">さらに表示</a>
```

- JS有効時は、無限スクロールまたは「さらに表示」操作でクライアント側の商品表示を追加する。
- 追加読み込み時にURLは変更しない。
- 初期HTMLには、SEOと非JS環境のため次ページへの実リンクを残す。
- ユーザー操作またはスクロールによって最後まで表示した後は、次ページリンクを消し、「すべての商品を表示しました」と表示する。
- JS初期化直後に全件を自動読み込みして、次ページリンクを即座に消す挙動は避ける。
- `/items/<category_slug>?page=2` 側にも、次ページがある場合は `/items/<category_slug>?page=3` への実リンクを出す。
- 最終ページには次ページリンクを出さない。

## ナビゲーション

- カテゴリは商品リストの上に横並びのリンクとして表示する。
- カテゴリリンクは実体を `a` タグにする。

```html
<a href="/items/spoons">スプーン</a>
```

- JS有効時はクリックを拾い、ページ遷移せずにURLと表示カテゴリをクライアント側で更新する。
- JS無効、クローラー、直アクセス時は、該当カテゴリのISR HTMLを返す。
- 現在カテゴリには `aria-current="page"` を付ける。

## データ取得

- Next.jsで実装する。
- 商品一覧、カテゴリ一覧、商品詳細はISRを使用する。
- 初期HTMLには、現在のURLに必要な商品データを含める。
  - `/items` は全商品一覧の1ページ目
  - `/items/spoons` はスプーンの1ページ目
  - `/item/<product_id>` は対象商品の詳細
- Hydration後、全商品のデータを別途取得し、IndexedDBに保存する。
- IndexedDBに全商品データがある場合、一覧の検索、絞り込み、並び替え、追加表示は通信せずクライアントで完結させる。
- IndexedDBにデータがない場合は、バックグラウンドで全商品データを取得して保存する。

## フィルタとソートの挙動

- カテゴリだけURLを持たせる。
- 検索、並び替え、ブランド、SALE、在庫状態、カテゴリスペックはURLを持たせない。
- 検索、並び替え、ブランド、SALE、在庫状態、カテゴリスペックは、IndexedDB上の商品データに対してクライアント側で処理する。
- これらの状態は一時的な表示条件として扱い、ページリロード時に初期状態へ戻る。
- カテゴリスペックのフィルタは、カテゴリが選択されていて、カテゴリ側のスペック定義がフィルタ対象になっている場合だけ表示する。

## 商品詳細から戻る導線

- 商品詳細にはBackボタンを表示する。
- Backボタンの実体は、商品のカテゴリ一覧へ戻る `a` タグにする。

```html
<a href="/items/spoons">Back</a>
```

- JS有効時、商品一覧から遷移してきた履歴がある場合だけブラウザの戻る挙動にする。
- 検索、並び替え、ブランド、SALEなどURLを持たない一覧状態を復元できるようにする。
- 直アクセス、外部サイト、検索エンジンから商品詳細へ来た場合は、`href` のカテゴリ一覧へ移動する。
- ブラウザ履歴でショップ外へ戻ってしまう挙動は避ける。

## パンくず

商品詳細では以下のパンくずを表示する。

```txt
トップ > 商品一覧 > スプーン > 商品名
```

リンク先は以下の通り。

```txt
トップ: /
商品一覧: /items
カテゴリ: /items/<category_slug>
商品名: 現在ページ
```

## index対象

- `/items`
- `/items?page=<page>`
- `/items/<category_slug>`
- `/items/<category_slug>?page=<page>`
- `/item/<product_id>`
- `/news`
- `/news/<news_id>`
- `/contact`
- `/law`
- `/privacy`

検索、並び替え、ブランド、SALE、在庫状態の組み合わせURLは作成せず、index対象にしない。
