# データ管理・レンダリング 概要（ショップ側）

> ざっくり方針だけ。詳細・根拠は [`データ管理.md`](./データ管理.md) を参照。

## 一言でいうと

初回はサーバーが商品データ入り HTML を返し（SEO・初速）、ハイドレーション後は全商品を IndexedDB に持つ**完全クライアント SPA**に切り替わる。遷移はサーバー往復ゼロ、鮮度は Realtime ＋ 復帰時同期で保つ。

## 骨子

- 初回 / 直リンク / クローラー … SSR の HTML（CDN キャッシュ）を即表示（スケルトンなし）。
- 以降の遷移 … IndexedDB（Dexie `liveQuery`）から描画。**サーバー往復ゼロ**。
- 全商品（1000 件未満）をブラウザから Supabase へ直接取得 → IndexedDB にミラー。
- 検索・絞り込み・並び替え・ページングはクライアントで完結。
- カート・お気に入りはクライアントのみ（JS 必須）。

## レンダリング方針

| ルート                                                          | 方式                                                |
| --------------------------------------------------------------- | --------------------------------------------------- |
| `/`・`/items…`・`/item/<id>`・`/news…`（動的）                  | SSR ＋ CDN キャッシュ（タグで無効化。ISR は不採用） |
| `/law`・`/privacy`・`/contact`・`/checkout/cancel`・404（静的） | prerender（SSG）                                    |
| `/checkout/success`                                             | SSR（キャッシュなし・noindex）                      |

- 全ルート `ssr = true` / `csr = true`。
- 動的ルートは `+page.ts`（universal load）。サーバー時のみ Supabase 取得＋`setHeaders` で CDN キャッシュ／タグ、ブラウザ時は Dexie。

## 鮮度の保ち方（2 系統）

- **実ユーザー**：IndexedDB ＋ Supabase Realtime（カタログ meta 購読）＋ ブラウザ復帰時同期で常に最新。
- **クローラー・非 JS**：管理変更／購入イベントで `dangerouslyDeleteByTag` → 次アクセスは再 SSR で最新（stale を返さない。時間ベース失効なし）。
