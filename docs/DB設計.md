## 自動採番

`自動採番` は Postgres の `GENERATED BY DEFAULT AS IDENTITY` を使用する。

## products

商品。

| カラム名            | 型          | NULL | デフォルト | 制約                              | 説明             |
| ------------------- | ----------- | ---: | ---------- | --------------------------------- | ---------------- |
| id                  | bigint      |   NO | 自動採番   | 主キー                            | 商品ID           |
| product_brand_id    | bigint      |  YES | -          | 外部キー（product_brands.id）     | ブランドID       |
| product_category_id | bigint      |  YES | -          | 外部キー（product_categories.id） | カテゴリID       |
| name                | text        |   NO | -          | 空文字不可                        | 商品名           |
| catch_phrase        | text        |  YES | -          | 空文字不可                        | キャッチフレーズ |
| description         | text        |  YES | -          | 空文字不可                        | 商品説明         |
| sku                 | text        |  YES | -          | 一意, 空文字不可                  | 自社管理コード   |
| jan_code            | text        |  YES | -          | 空文字不可                        | JANコード        |
| regular_price       | integer     |   NO | -          | 1以上                             | 通常価格         |
| sale_price          | integer     |  YES | -          | 1以上                             | セール価格       |
| stock_quantity      | integer     |   NO | 0          | 0以上                             | 在庫数           |
| sort_order          | integer     |   NO | 0          | 一意, 0以上                       | 表示順           |
| new_badge_mode      | text        |   NO | 'auto'     | `auto` / `show` / `hide` のみ     | NEWタグ表示      |
| is_published        | boolean     |   NO | false      | -                                 | 公開状態         |
| created_at          | timestamptz |   NO | now()      | -                                 | 作成日時         |
| updated_at          | timestamptz |   NO | now()      | -                                 | 更新日時         |

テーブル制約:

- セール価格を設定する場合は、通常価格より低くする。

DB以外で保証するルール:

- 商品カテゴリが未設定の場合は、商品ブランドも未設定にする。
- 商品カテゴリと商品ブランドを両方設定する場合は、`product_category_brands` に存在する組み合わせだけ使用する。
- `new_badge_mode` が `auto` の場合は、商品作成日に応じてNEWタグの表示有無を判定する。
- `new_badge_mode` が `show` の場合はNEWタグを表示し、`hide` の場合は表示しない。

## product_images

商品ごとの画像。

| カラム名   | 型          | NULL | デフォルト | 制約                    | 説明              |
| ---------- | ----------- | ---: | ---------- | ----------------------- | ----------------- |
| id         | bigint      |   NO | 自動採番   | 主キー                  | 商品画像ID        |
| product_id | bigint      |   NO | -          | 外部キー（products.id） | 商品ID            |
| image_url  | text        |   NO | -          | 空文字不可              | 画像URL（R2想定） |
| sort_order | integer     |   NO | 0          | 0以上                   | 表示順            |
| created_at | timestamptz |   NO | now()      | -                       | 作成日時          |
| updated_at | timestamptz |   NO | now()      | -                       | 更新日時          |

テーブル制約:

- 同じ商品内で、表示順は重複不可。

DB以外で保証するルール:

- 画像がある場合は、`sort_order = 0` の画像を一覧表示用のメイン画像として扱う。
- 画像がない商品は、UI側で画像なし表示にする。

| ------------------- | ----------- | ---: | ---------- | --------------------------------- | ------------------------------------- |
| id                  | bigint      |   NO | 自動採番   | 主キー                            | スペック定義ID                        |
| product_category_id | bigint      |   NO | -          | 外部キー（product_categories.id） | カテゴリID                            |
| slug                | text        |   NO | -          | 空文字不可                        | URLクエリと内部識別に使用するスラッグ |
| label               | text        |   NO | -          | 空文字不可                        | 表示名                                |
| value_type          | text        |   NO | 'text'     | `text` / `number` / `option` のみ | 値の種類                              |
| unit                | text        |  YES | -          | 空文字不可                        | 単位                                  |
| is_required         | boolean     |   NO | false      | -                                 | 入力必須                              |
| is_filterable       | boolean     |   NO | false      | -                                 | ショップ側の商品一覧フィルタ対象      |

- 同じカテゴリ内で、表示順は重複不可。

## category_spec_options

カテゴリスペックごとの選択肢。値の種類が `option` のスペック定義で使用する。

| カラム名                    | 型          | NULL | デフォルト | 制約                                     | 説明                            |
| --------------------------- | ----------- | ---: | ---------- | ---------------------------------------- | ------------------------------- |
| id                          | bigint      |   NO | 自動採番   | 主キー                                   | 選択肢ID                        |
| category_spec_definition_id | bigint      |   NO | -          | 外部キー（category_spec_definitions.id） | スペック定義ID                  |
| label                       | text        |   NO | -          | 空文字不可                               | 表示名                          |
| slug                        | text        |  YES | -          | 空文字不可                               | URLクエリ値（フィルタ対象のみ） |
| sort_order                  | integer     |   NO | 0          | 0以上                                    | 表示順                          |
| created_at                  | timestamptz |   NO | now()      | -                                        | 作成日時                        |
| updated_at                  | timestamptz |   NO | now()      | -                                        | 更新日時                        |

テーブル制約:

- 同じスペック定義内で、表示名は重複不可。
- 同じスペック定義内で、スラッグがある場合は重複不可。
- 同じスペック定義内で、表示順は重複不可。

DB以外で保証するルール:

- 選択肢は、値の種類が `option` のスペック定義にだけ登録する。
- 同じスペック定義内で、スラッグあり・なしを混在させない。
- 商品一覧フィルタ対象のスペック定義に紐づく選択肢は、すべてスラッグを設定する。
- 商品一覧フィルタ対象ではないスペック定義に紐づく選択肢は、スラッグを設定しない。

## product_spec_values

商品ごとのカテゴリスペック値。

| カラム名                    | 型            | NULL | デフォルト | 制約                                     | 説明           |
| --------------------------- | ------------- | ---: | ---------- | ---------------------------------------- | -------------- |
| id                          | bigint        |   NO | 自動採番   | 主キー                                   | スペック値ID   |
| product_id                  | bigint        |   NO | -          | 外部キー（products.id）                  | 商品ID         |
| category_spec_definition_id | bigint        |   NO | -          | 外部キー（category_spec_definitions.id） | スペック定義ID |
| category_spec_option_id     | bigint        |  YES | -          | 外部キー（category_spec_options.id）     | 選択肢ID       |
| value_text                  | text          |  YES | -          | 空文字不可                               | テキスト値     |
| value_number                | numeric(12,4) |  YES | -          | -                                        | 数値           |
| created_at                  | timestamptz   |   NO | now()      | -                                        | 作成日時       |
| updated_at                  | timestamptz   |   NO | now()      | -                                        | 更新日時       |

テーブル制約:

- 同じ商品に対して、同じスペック定義は1件のみ登録可能。
- テキスト値、数値、選択肢IDはどれか一方だけ入力する。複数入力・全未入力は不可。

DB以外で保証するルール:

- `category_spec_definition_id` は、対象商品の `product_category_id` と同じカテゴリのスペック定義のみ使用する。
- 値の種類が `text` のスペックは、テキスト値を使用する。
- 値の種類が `number` のスペックは、数値を使用する。
- 値の種類が `option` のスペックは、選択肢IDを使用する。
- `category_spec_option_id` を使う場合、その選択肢は同じ `category_spec_definition_id` に紐づくものだけ使用する。
- 商品一覧フィルタ対象にする定型値は、原則として値の種類を `option` にする。
