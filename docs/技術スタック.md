# 技術スタック

このプロジェクト（skymtackle リニューアル / 現状はデザイン案＝フロントUIのみ）の採用技術。
仕様変更前の旧スタック（Next.js 前提）は `docs/old/技術スタック.md` を参照。

## 採用スタック（確定分）

| 領域                 | 採用技術                                               | 用途・補足                                                                              |
| -------------------- | ------------------------------------------------------ | --------------------------------------------------------------------------------------- |
| アプリケーション基盤 | SvelteKit 2 + Svelte 5（runes）                        | 旧 Next.js App Router から変更。初期 SSR → 以降クライアント SPA                         |
| 言語                 | TypeScript                                             |                                                                                         |
| ホスティング         | Vercel                                                 | `@sveltejs/adapter-vercel` を使用。商品ページ等は SSR ＋ CDN キャッシュ（タグで無効化） |
| スタイリング         | Tailwind CSS v4 + CSS変数                              | デザイン値は CSS 変数で管理。`dark:` は多用しない                                       |
| UIコンポーネント     | shadcn-svelte + Bits UI                                | 旧 shadcn/ui + Radix UI の移行先。アクセシビリティを担保                                |
| アイコン             | lucide-svelte                                          | 旧 lucide-react から                                                                    |
| ローカルDB           | IndexedDB + Dexie.js                                   | 全商品をキャッシュ。`liveQuery` で UI へリアクティブ反映                                |
| ↳ Svelte 連携        | dexie-svelte-query（`stateQuery`）                     | `current` / `isLoading` でスケルトン制御。SSR セーフ                                    |
| データベース         | Supabase Database / Postgres                           |                                                                                         |
| ブラウザ→DBアクセス  | Supabase JS（`@supabase/ssr`、anon / publishable key） | ブラウザから直接、全商品を取得 → IndexedDB                                              |
| リアルタイム         | Supabase Realtime（`postgres_changes`）                | カタログ用 meta テーブルを購読。`worker: true` + `heartbeatCallback`                    |
| 認可                 | Postgres Row Level Security                            | anon ロールに SELECT 許可                                                               |
| 決済                 | Stripe Checkout hosted                                 | 配送先・送料は Stripe 側。`/checkout/success`・`/checkout/cancel`                       |

データ取得・同期・レンダリングの詳細は `docs/ショップ側/データ管理.md` を参照。
