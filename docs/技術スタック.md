# 技術スタック

## 採用スタック

| 領域                 | 採用技術                     | 用途                                                                              |
| -------------------- | ---------------------------- | --------------------------------------------------------------------------------- |
| アプリケーション基盤 | Next.js App Router           | ショップ側の静的生成/ISR、管理側SPAの配信、Server Actions、Route Handlers         |
| UIランタイム         | React                        | Server ComponentsとClient Componentsを使い分ける                                  |
| 言語                 | TypeScript                   | アプリケーション、Server Actions、Supabase型定義を一貫して型安全に扱う            |
| ホスティング         | Vercel                       | Next.jsのデプロイ、ISR、Functions、プレビュー環境、環境変数管理に使用する         |
| スタイリング         | Tailwind CSS v4 + CSS変数    | デザイン値をCSS変数で管理し、ユーティリティクラスでUIを組む                       |
| UIコンポーネント     | shadcn/ui + Radix UI         | アクセシビリティを担保したUI部品の土台にする                                      |
| アイコン             | lucide-react                 | 管理画面やショップUIの操作アイコンに使用する                                      |
| フォーム             | React Hook Form              | 管理画面の商品、ニュース、設定フォームを軽量に扱う                                |
| validation           | Zod                          | Server Actions、フォーム、外部入力のスキーマ検証に使用する                        |
| サーバー状態管理     | TanStack Query               | 管理側の取得、再取得、range変更、エラー、ローディング状態を管理する               |
| ローカルDB           | IndexedDB + Dexie.js         | 商品一覧、管理側一覧、meta情報などのブラウザ内キャッシュを扱う                    |
| データベース         | Supabase Database / Postgres | 商品、カテゴリ、ブランド、注文、お問い合わせ、顧客、管理ログを保存する            |
| ORM / DBアクセス     | Prisma Client                | Server ActionsやRoute HandlersからPostgresへ型安全にアクセスする                  |
| 認証                 | Supabase Auth + Google OAuth | 管理画面のGoogleログインに使用する                                                |
| 認可                 | Postgres Row Level Security  | 公開データ、管理データ、管理者権限のアクセス範囲をDB側で制御する                  |
| ファイル管理         | Cloudflare R2                | 商品画像、ニュース画像などをS3互換APIで保存する                                   |
| 決済                 | Stripe Checkout              | Stripeで有効化する決済手段とCheckout Session作成に使用する                        |
| Webhook受信          | Route Handlers               | Stripeの支払い完了通知など外部サービスイベントを受け取り、後続処理を行う          |
| メール送信           | Resend                       | 注文確認、問い合わせ受付、問い合わせ返信、管理者通知に使用する                    |
| 外部通知             | Slack Incoming Webhooks      | 注文、問い合わせ、重要な管理操作、エラーなどを運用者へ通知する                    |
| リアルタイム通知     | Supabase Realtime            | 管理画面で注文や管理ログの更新通知を受け取る                                      |
| 通知UI               | Sonner                       | 管理画面内のtoast通知に使用する                                                   |
| 監視                 | Sentry                       | フロントエラー、Server Actions、Route Handlers、Webhookの監視に使用する           |
| E2E/APIテスト        | Playwright                   | ショップ側E2E、管理側E2E、Route HandlersやWebhookのAPIテストに使用する            |
| DBスキーマ管理       | Supabase CLI migrations      | DB変更、RLS、function、triggerをSQL migrationで管理する。Prisma Migrateは使わない |
