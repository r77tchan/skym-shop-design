# AIへの指示

1. 編集や変更は明示的な指示があった場合にのみ行うこと。気を利かせた対応は有り難いのだが、まだ編集や変更に進んでほしくないことがあるので、GOサインを待つこと。
2. ただ動けばいい訳ではない。コードが読みやすく、保守性の高い、最適なプロジェクトを目指すこと。
3. AIの学習データには古い情報もある。必ず正しい情報であることの確認と、最適な実装方法を調査してから、実装に進むこと。
4. 最適なプロジェクトを目指したいので、修正変更の際には最小 diff に拘る必要はない。
5. 作業後には、タイプチェック、リント、プロジェクト全体へのフォーマットを必要に応じて行うこと。（必須ではなく、判断は任せます）
